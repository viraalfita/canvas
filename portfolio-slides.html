Canvas Studio.

A visual AI workflow generator — prompt to get image and video, all flowing through one node graph instead of five separate tools.

Year2026
RoleFull-Stack Dev

Concept

AI content production usually means juggling three tabs and five tools — Canvas folds the whole pipeline into one graph where prompts become images become videos, node to node.

The Story

Before Canvas, the team's AI content workflow was spread across multiple tools — prompts written in ChatGPT, images generated in Magnific, videos created in Runway, and final edits assembled manually in Premiere. Moving between platforms constantly broke context, slowed iteration, and made the creative process difficult to manage efficiently. To solve this, I redesigned the workflow into a unified node-based system, where each stage — Text Prompt, Image, Video, and Scene Composer — connects seamlessly inside a single interactive canvas.

I built the platform using Next.js 16, React Flow, and Supabase, with serverless execution on Vercel and async polling for AI image and video generation through APImart. As the team continued using the platform, new features naturally evolved from real creative needs, including AI-powered prompt enhancement, per-node output history for easier iteration, and an in-browser Scene Composer powered by ffmpeg.wasm to avoid unnecessary re-uploads.

This project taught me that creative tools should reduce friction and make experimentation feel effortless. The easier it is for people to test, revise, and explore ideas, the more likely they are to stay engaged in the creative process.
